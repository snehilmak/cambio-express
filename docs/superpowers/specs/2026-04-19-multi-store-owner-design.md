# Multi-Store Owner Design

## Goal

Allow a single owner to manage multiple store locations under one login — with an aggregate dashboard showing combined stats and expandable per-store summaries, and an invite-code system so store admins can grant an owner access to their store.

## Architecture

Add an `"owner"` role to the existing User model (no schema change — `role` is already a string, `store_id` is already nullable). Two new tables handle the many-to-many link between owners and stores, and the invite code lifecycle. Owner accounts are view-only: they see data but cannot create transfers, batches, or reports. Each store retains its own admin and employees.

**Tech Stack:** Flask, SQLAlchemy, Jinja2, pytest-flask, in-memory SQLite for tests.

---

## Data Model

### New: `StoreOwnerLink`

Links an owner user to a store.

| Column | Type | Notes |
|---|---|---|
| `id` | Integer PK | |
| `owner_id` | Integer FK → User.id | must have role="owner" |
| `store_id` | Integer FK → Store.id | |
| `linked_at` | DateTime | default utcnow |
| UniqueConstraint(owner_id, store_id) | | prevents duplicate links |

### New: `OwnerInviteCode`

Codes generated by store admins for owners to redeem.

| Column | Type | Notes |
|---|---|---|
| `id` | Integer PK | |
| `store_id` | Integer FK → Store.id | which store this code unlocks |
| `code` | String(8), unique | random uppercase alphanumeric |
| `created_by` | Integer FK → User.id | admin who generated it |
| `created_at` | DateTime | default utcnow |
| `expires_at` | DateTime | 7 days from created_at |
| `used_at` | DateTime, nullable | null = not yet redeemed |
| `used_by_owner_id` | Integer FK → User.id, nullable | owner who redeemed it |

**No changes to existing models.** `User.role` already supports new string values. `User.store_id` is already nullable (owners have `store_id=None`, same as superadmin).

## Owner Signup (`/signup/owner`)

1. Form fields: Full Name, Email, Password
2. Validation: email required + valid format + not already taken by any User (admin or owner), password ≥ 8 chars
3. On success: create `User(role="owner", store_id=None, username=email, full_name=full_name)`
4. Set session (`user_id`, `role="owner"`, `store_id=None`), redirect to `/owner/dashboard`
5. The existing `/signup` page gets a small footer link: *"Own multiple locations? Sign up as an owner →"* pointing to `/signup/owner`

---

## New `owner_required` Decorator

```python
def owner_required(f):
    @wraps(f)
    def d(*a, **k):
        if "user_id" not in session:
            return redirect(url_for("login"))
        u = current_user()
        if not u or u.role != "owner":
            abort(403)
        return f(*a, **k)
    return d
```

Owners are added to `_TRIAL_EXEMPT` (they have no store, no trial to enforce).

## Owner Dashboard (`/owner/dashboard`)

### Aggregate Stats (top of page)

Four stat cards using existing `.stat-card` CSS class:
- **Transfers Today** — count of all Transfer rows across all linked stores where `send_date == today`
- **Volume Today** — sum of `send_amount` across those transfers
- **Stores** — count of linked stores
- **Over/Short Today** — sum of `over_short` from DailyReport rows for today across all stores

### Date Filter

Dropdown or tab: **Today / This Month / This Year**. Stored as a `?period=` query param (default `today`). All per-store card data respects this filter.

### Store Cards (below stats)

One card per linked store, ordered by store name. Each card shows:
- Store name + plan badge (trial/basic/pro)
- For the selected period:
  - Transfer count + total $ volume
  - Daily report: total receipts + over/short (if a report exists for the period)
- "Unlink Store" button (POSTs to `/owner/unlink/<store_id>`)

If no stores linked: a prompt card — *"Enter an invite code from your store manager to connect your first store."* with a text input and Submit button (POSTs to `/owner/link`).

Code entry form is always shown at the bottom even when stores are linked (to add more).

## Admin Settings — Owner Access Tab

New 4th tab in `/admin/settings`: **Owner Access**

### State: No owner linked, no active code

Shows "Generate Invite Code" button. POST to `/admin/settings/owner/generate-code`:
1. Invalidate any existing unused, unexpired codes for this store (set `expires_at = utcnow()`)
2. Generate new 8-char uppercase alphanumeric code using `secrets.token_hex`
3. Create `OwnerInviteCode(store_id, code, created_by, expires_at=utcnow()+7days)`
4. Redirect back to Owner Access tab

### State: Active unused code exists

Shows the code in a read-only input with a Copy button. Shows expiry date. Shows "Generate New Code" button (invalidates existing, creates new).

### State: Code was redeemed — owner is linked

Shows: *"[Owner Name] ([email]) has access to this store."* with a "Remove Access" button. POST to `/admin/settings/owner/remove-access`:
1. Delete the `StoreOwnerLink` row for this store
2. Flash success, redirect back to Owner Access tab

## Owner Link Flow (`/owner/link`)

POST with `code` form field:
1. Look up `OwnerInviteCode` where `code == submitted_code` AND `used_at IS NULL` AND `expires_at > utcnow()`
2. If not found: flash *"Invalid or expired code."*, redirect back to `/owner/dashboard`
3. If already linked to this owner: flash *"You're already connected to this store."*, redirect
4. Create `StoreOwnerLink(owner_id=current_user().id, store_id=invite.store_id)`
5. Mark `invite.used_at = utcnow()`, `invite.used_by_owner_id = current_user().id`
6. Commit, flash *"[Store Name] connected successfully."*, redirect to `/owner/dashboard`

---

## Owner Unlink Flow (`/owner/unlink/<store_id>`)

POST:
1. Find `StoreOwnerLink` where `owner_id == current_user().id` AND `store_id == store_id` → 404 if not found
2. Delete the row, commit
3. Flash *"Store removed from your account."*, redirect to `/owner/dashboard`

## Testing

### Owner Signup
- Valid signup creates owner user with role="owner", store_id=None
- Duplicate email rejected
- Short password rejected
- Redirects to /owner/dashboard on success

### Owner Dashboard
- Loads with no stores linked (shows prompt)
- Shows aggregate stats after linking a store
- Period filter changes the stats displayed

### Invite Code — Admin Side
- Admin can generate a code (stored in DB, 7-day expiry)
- Generating a new code invalidates the previous unused one
- Owner Access tab shows correct state for each scenario

### Invite Code — Owner Side
- Valid code links owner to store
- Expired code rejected
- Already-used code rejected
- Invalid code rejected
- Owner already linked to that store is handled gracefully

### Unlink
- Owner can unlink a store
- Cannot unlink a store they're not linked to (404)
